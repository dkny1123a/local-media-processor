## 📊 API 端点对比

### 同步端点（原有）

| 端点 | 功能 | 响应时间 | 状态 |
|------|------|---------|------|
| `/api/audio/process` | 音频处理 | 30-300秒 | 阻塞 |
| `/api/audio/transcribe` | 语音识别 | 60-600秒 | 阻塞 |
| `/api/image/upscale` | 图像放大 | 10-120秒 | 阻塞 |

### 异步端点（新增）

| 端点 | 功能 | 响应时间 | 状态 |
|------|------|---------|------|
| `/async/audio/process` | 音频处理 | <1秒 | 异步 |
| `/async/audio/transcribe` | 语音识别 | <1秒 | 异步 |
| `/async/image/upscale` | 图像放大 | <1秒 | 异步 |
| `/async/tasks/{id}` | 任务状态查询 | <100ms | 实时 |

## 🎯 使用建议

### 场景 1: 大文件处理

**推荐**: 使用异步端点

```bash
# 提交任务
curl -X POST http://localhost:8000/async/audio/process \
  -F "file=@large_video.mp4"

# 定期查询进度
curl http://localhost:8000/async/tasks/{task_id}

# 任务完成后下载结果
curl http://localhost:8000/api/files/download/{output_file}
```

## 🐛 常见问题

### Q1: 任务一直处于 PENDING 状态

**原因**: Celery Worker 未启动或连接失败

**解决方案**:
```bash
# 检查 Worker 状态
docker-compose ps celery-worker

# 检查 Worker 日志
docker-compose logs celery-worker

# 重启 Worker
docker-compose restart celery-worker
```

---

### Q2: Redis 连接失败

**原因**: Redis 服务未启动

**解决方案**:
```bash
# 检查 Redis 状态
docker-compose ps redis

# 测试 Redis 连接
docker exec -it media-redis redis-cli ping

# 重启 Redis
docker-compose restart redis
```

---

### Q3: 内存占用过高

**原因**: 模型未自动释放

**解决方案**:
```bash
# 查看内存统计
curl http://localhost:8000/api/system/stats

# 查看模型状态
curl http://localhost:8000/api/system/stats | jq '.models'

# 手动释放模型（需要添加端点）
# 或等待自动释放（5分钟后）
```
